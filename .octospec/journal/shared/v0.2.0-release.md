# Journal: v0.2.0-release

> Pure release收口 for moaray **v0.2.0**. The feature (v0.2-P1 persistent
> usage/cost accounting) was already merged to `main` (13ab87b, PR #13) and passed
> E2E regression (6 checkpoints, YUJ-6037). This PR只动 version + docs + octospec
> Finish — no `.rs` touched.

## What was done
- **Version bump `0.1.0 → 0.2.0`**: only the workspace `Cargo.toml`
  `[workspace.package] version` needed editing — every crate already uses
  `version.workspace = true` (grep confirmed no per-crate version strings). `cargo
  build` propagated the bump into `Cargo.lock`; all six moaray crates
  (moaray, -core, -config, -moa, -providers, -store) + mock-upstream now read
  `0.2.0`.
- **RELEASE.md**: added a v0.2.0 section on top, v0.1.0 history kept below. Section
  states the new capability (persistent per-arm request store + cost accounting:
  `UsageSink` trait, `moaray-store` crate, `SqliteSink` on WAL, dedicated OS-thread
  writer, nano-USD cost), the **posture** (best-effort telemetry-grade — NOT an
  invoice-grade ledger; drop-not-block; raw token + price snapshot ⇒ recomputable),
  the acceptance gate, the default-off config (`server.usage_store` absent ⇒
  `NullSink`, zero overhead, restart-frozen), and known limitations (streaming
  passthrough unaccounted but observable via `moaray_usage_unaccounted_stream_total`;
  load-smoke baseline fix tracked in #14; invoice-grade durability deferred to
  v0.2-P2).
- **DESIGN.md §5**: no-op — PR #13 already changed it from "out of scope for v1" to
  "landed in v0.2-P1". Verified, left untouched.

## Learning / gotcha
- **Single source of truth for version.** Because every crate inherits
  `version.workspace = true`, the bump is a one-line edit + a `cargo build` to sync
  `Cargo.lock`. Grepping for stray per-crate `version = "0.1.0"` strings before
  trusting that (there were none) is the cheap insurance against a half-bumped
  workspace.
- **Don't overclaim in RELEASE.md.** The accounting store is deliberately
  telemetry-grade (drop-not-block), so the release notes must say "NOT an
  invoice-grade ledger" out loud — otherwise a reader assumes durable billing and
  is surprised by dropped rows under overload. The posture line is load-bearing
  documentation, not marketing.
