## What was done
- **moaray-core**: `UsageSink` trait + `UsageRecord` DTO (all-`Send`, no secret
  fields) + the pure `compute_cost` helper (nano-USD, i128 intermediates,
  round-half-up, `None` when tokens/price absent). `RawResponse` gained
  `usage: Option<(i64,i64)>`, tapped INSIDE `collect_response` while the bytes are
  still held (no second drain → `streaming-passthrough` honored).
- **moaray-store** (new crate): `SqliteSink` (WAL, synchronous=NORMAL,
  user_version=1) writing via a **dedicated OS thread** draining a `crossbeam`
  bounded channel (NOT tokio — rusqlite is sync). `record` = `try_send` only;
  full/disconnected channel drops + bumps `moaray_usage_dropped_total`.
  `UsageWriterHandle::flush_and_join(timeout)` (detach-on-timeout). Plus `NullSink`
  (default) + `VecSink` (test util).
- **moaray-moa**: `run()` now returns `Result<MoaRun{proposers, aggregator:Option,
  outcome:Result}>` so EVERY return after fan-out carries the arm outcomes — quorum
  failure, the aggregator-resolve `?` at orchestrator.rs:279, aggregator
  timeout/error. Pre-fan-out ModelNotFound stays a plain Err (exempt). `ArmOutcome`
  enriched with prompt/completion tokens. Blast radius held to the 5 named sites.
- **moaray-config**: per-model `price_{prompt,completion}_per_mtok_usd` (USD/Mtok)
  → integer nano-USD/Mtok at validate (rejects negative/non-finite); prices are
  hot-reloadable. `server.usage_store {path, channel_capacity, batch_size}` —
  absent ⇒ NullSink. usage_store is restart-frozen (reload warns + keeps running
  store, via the pure `frozen_field_changes` helper).
- **moaray (bin)**: `AppState.usage_sink` + `AppState::with_sink` injection seam.
  `main()` builds (sink, writer_handle); shutdown refactored — signal future
  resolves immediately, `flush_and_join` runs AFTER `serve().await` returns; the
  dead post-serve `sleep(grace)` was deleted. App emission seam: non-stream
  passthrough + every MoA arm book a row (priced from the routed snapshot, codex
  C1); per-arm metrics moved to fire unconditionally (lockstep with rows on the
  error path); streaming bumps `moaray_usage_unaccounted_stream_total`.

## Learning / gotcha
- **The injection seam (`AppState::with_sink`) is what makes the gates real.**
  Without it the default `NullSink` is a no-op and every "rows landed" assertion
  passes on an empty set. This was R1 P1-① and it is the load-bearing test-design
  fix, not a code nicety.
- **`run()`'s `?` at orchestrator.rs:279 silently dropped a whole class of
  billable rows.** The aggregator-resolve happens after fan-out + quorum pass, so
  proposers already incurred cost; routing it through `MoaRun` (not `?`) is the
  difference between billing a failed run and losing it. Kept `run()` returning
  `Result<MoaRun>` (not bare `MoaRun`) so the truly-zero-cost pre-fan-out
  ModelNotFound stays a plain Err — trace.rs:129 became `.unwrap().outcome.unwrap()`.
- **Tap usage INSIDE `collect_response`, never in the caller.** By the time
  `collect_response` returns, the bytes are already moved into the relay stream;
  parsing in the caller would force a second drain (a plan STOP condition).
- **Anthropic erases usage-absent to `0,0`** in `anthropic_to_openai`. Judge
  absence on the RAW upstream `usage` key before translation → `ok_no_usage` when
  truly absent, without ever rewriting genuinely-zero openai rows.
- **governed.rs is in the `moaray` BIN crate**, not `moaray-providers` (the plan
  said providers); the 3 RawResponse mock sites at 379/390/752 still matched.
